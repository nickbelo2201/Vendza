# Plano de Execucao Amanha

## Meta do dia

Sair da documentacao consolidada para o bootstrap real do produto.

## Ordem recomendada

1. Entrar nesta pasta `app`
2. Criar o monorepo e os manifests base
3. Definir schema do banco
4. Subir ambiente local
5. Implementar auth, tenant e loja
6. Implementar catalogo, estoque e board de pedidos

## Checklist pratico

- ler `02-prd-v1.md`
- ler `07-arquitetura-tecnica.md`
- transformar esta estrutura em repositorio real
- criar tabela de `stores`
- criar tabela de `products`
- criar tabela de `orders`
- criar `order_events`
- fechar contrato basico da API
- montar layout base de `web-client`
- montar layout base de `web-partner`

## Entrega minima do primeiro dia

- repo inicializado
- estrutura de apps criada
- banco modelado
- auth decidido
- primeira rota de health e primeira pagina de layout prontas

## Regra de foco

Nao reabrir analise competitiva durante a execucao do bootstrap.

Agora o problema deixou de ser descobrir o que fazer. O problema passa a ser construir na ordem certa.
